Secure HTTPS Server

Role: Back-end Developer â€¢ Tools: Node.js, Express, TLS/SSL, Passport Google OAuth â€¢ Year: 2025

Problem

Modern web applications require secure authentication, encrypted communication, and reliable session handling. For this project, the challenge was to build a fully secure HTTPS server using Node.js and Express, implement Google OAuth 2.0, and resolve common auth issues such as incorrect callback handling, redirect mismatches, and improperly configured TLS certificates.

Approach & Process

  1. HTTPS Configuration: Implemented TLS/SSL using local certificates, configured secure cipher suites, enforced HTTPS-only traffic, and enabled HTTP Strict Transport Security (HSTS) to prevent downgrade attacks.
  2. OAuth 2.0 Integration (Google): Set up Passport.js with Google Strategy, added explicit scopes for profile and email, and resolved redirect_uri_mismatch by properly syncing callback URLs across Google Cloud Console, env files, and Express routes.
  3. Session & Cookie Security: Implemented secure, HttpOnly session cookies, validated tokens, and added middleware for authentication checks.
  4. Error Handling & Diagnostics: Fixed issues like Missing required parameter: scope, handled expired tokens gracefully, and improved logging for OAuth debugging and certificate-loading errors.
  5. Documentation: Created a clean developer-friendly setup guide covering certificates, .env configuration, routing structure, and debugging steps for OAuth flows.

Outcome

The final server provides a fully encrypted authentication workflow with strong security practices. The Google OAuth login runs smoothly with predictable redirects and a consistent account picker prompt. The routing and middleware structure is modular and reusable, making the codebase easy to extend for future improvements.

Key Learnings

  • How TLS/SSL certificates interact with Express and local environments.
  • The importance of precise redirect URIs in OAuth authentication flows.
  • How to configure secure cookies and sessions for production-level security.
  • Better debugging practices for OAuth and authentication middleware.
